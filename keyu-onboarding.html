다섯 단계면 폰이 입력장치가 됩니다

PC에 서버를, 폰에 앱을 설치하고 PIN 한 번으로 페어링하면 끝입니다. 이후에는 앱을 열 때마다 저장된 인증키로 자동 재연결됩니다.

약 5분 소요 · 계정 불필요 · 클라우드 없음 · 페어링은 최초 한 번

1
Step 1 · PC

서버 설치

입력을 받을 PC에 KeyU 서버를 설치합니다. 서버는 트레이(메뉴바)에 조용히 상주하며, 창을 닫아도 종료되지 않습니다.

macOS

  1. .dmg를 열고 앱을 응용 프로그램으로 드래그합니다.
  2. Gatekeeper 경고가 표시되면 우클릭 → 열기로 첫 실행을 승인합니다.
  3. 안내에 따라 시스템 설정 → 개인정보 보호 및 보안 → 손쉬운 사용에서 KeyU를 허용합니다.

손쉬운 사용은 키·커서 입력에 필요한 유일한 권한입니다. 허용 전에는 입력이 무시되고 경고 배너가 표시됩니다.

Windows

  1. 인스톨러(setup.exe)를 실행합니다.
  2. 최초 실행 시 방화벽 인바운드 허용을 승인합니다.
  3. 별도 권한은 없습니다 — 설치 즉시 입력할 수 있습니다.

WebView2 런타임 필요(Windows 10/11 대부분 내장) · Bonjour 등 별도 프로그램 설치 불필요.

이 단계가 끝나면 — 트레이(메뉴바)에 KeyU 아이콘이 보입니다.
2
Step 2 · Phone

앱 설치

폰과 PC가 같은 Wi-Fi에 있는지 확인하고 앱을 설치합니다. 요청 권한은 로컬 네트워크(서버 발견)와 카메라(QR 스캔) 둘뿐입니다.

iOS · iPadOS

  • iOS / iPadOS 16.0 이상, iPhone·iPad 지원
  • 첫 실행 시 로컬 네트워크 권한을 허용해야 서버가 목록에 나타납니다.
  • iPad는 전 방향 회전과 가로 2-pane 레이아웃을 지원합니다.

App Store에서 받기 — 카메라로 스캔하거나

스토어에서 열기

Android

  • Android 8.0 (API 26) 이상
  • iOS와 동일한 입력 모드·편집 기능을 제공합니다.
  • QR을 스캔하면 딥링크로 페어링 화면이 바로 열립니다.

Google Play에서 받기 — 카메라로 스캔하거나

스토어에서 열기
이 단계가 끝나면 — 앱 첫 화면의 서버 목록에 내 PC 이름이 자동으로 나타납니다.
3
Step 3 · 최초 1회

페어링

처음 한 번만 서로를 승인합니다. 이후에는 이 과정이 다시 필요 없습니다.

  • 1

    서버에서 페어링 시작

    트레이 메뉴 또는 창의 페어링 탭에서 시작하면 6자리 PIN과 QR 코드가 함께 표시됩니다. 유효시간은 120초이고, 만료되면 자동으로 재생성됩니다.

  • 2

    앱에서 서버 선택

    목록에서 내 PC를 탭합니다. 목록에 없으면 서버 창에 표시된 주소:포트를 직접 입력할 수 있습니다.

  • 3

    PIN 입력 또는 QR 스캔

    PIN 6자리를 입력하거나 카메라로 QR을 스캔합니다. PIN은 평문으로 전송되지 않으며, 5회 실패 시 재생성됩니다.

  • 4

    이후는 자동

    서버가 발급한 인증키가 폰의 보안 저장소에 보관되어, 다음부터는 앱을 열면 자동으로 재연결됩니다.

이 단계가 끝나면 — 상단 상태 도트가 연결됨으로 바뀝니다. 페어링은 다시 없습니다.
4
Step 4 · 사용

입력 시작

하단 탭에서 입력 모드를 고르면 바로 PC에 입력됩니다. 커서는 손가락을 지연 없이 따라옵니다.

텐키

  • 숫자·연산·엔터 전용 패드 — 노트북에 없는 숫자패드를 대신합니다.
  • 길게 누르면 반복 입력됩니다.
  • 키 진동은 설정에서 끔·약·보통·강 4단계로 조절합니다.

트랙패드

제스처동작
1지 이동커서 이동
1지 탭좌클릭
2지 이동스크롤 (자연 방향)
2지 정지 + 왼쪽 리프트우클릭

iPhone 전용 — iPad에서는 트랙패드 모드가 비활성화됩니다.

버튼 그리드

  • 3×4 / 3×5 그리드의 버튼에 키·조합키·텍스트·매크로를 바인딩합니다.
  • Windows 서버에 연결하면 ⌘가 Ctrl로 자동 치환되고 표기도 바뀝니다.

시뮬레이션 바

  • 복사·붙여넣기 같은 자주 쓰는 액션을 가로 바로 배치합니다.
  • 켜 두면 어떤 입력 모드에서도 항상 위에 고정됩니다.
이 단계가 끝나면 — 폰에서 누른 키가 PC에 그대로 입력됩니다.
5
Step 5 · 나만의 레이아웃

커스터마이즈

쓰다 보면 손에 맞게 바꾸고 싶어집니다. 편집은 전부 앱 안에서 끝납니다.

  • A

    버튼 편집

    버튼을 추가·삭제·이동하고 라벨(⌘C 같은 조합 기호)과 색상을 지정합니다.

  • B

    액션 편집

    단일 키, 조합키(⌘⇧…), 텍스트 입력, 매크로 참조 — 버튼이 할 일을 정의합니다.

  • C

    매크로 편집

    키 입력·지연·텍스트의 시퀀스를 만들고 순서를 바꾸고, 보내기 전에 테스트 실행까지.

  • D

    프리셋 전환

    코딩·디자인·미디어 등 용도별 레이아웃을 여러 개 저장해 두고 골라 씁니다.

  • E

    서버 여러 대

    PC를 여러 대 페어링해 두면 상단 이름 배지 탭으로 즉시 전환됩니다. 연결이 끊기면 자동 페일오버합니다.

여기까지 오면 — KeyU가 내 작업 흐름에 맞춘 전용 입력장치가 됩니다.
Troubleshooting

문제 해결

서버가 목록에 나타나지 않아요

폰과 PC가 같은 Wi-Fi에 있는지, 앱의 로컬 네트워크 권한을 허용했는지 확인하세요. 게스트 Wi-Fi나 AP 격리처럼 기기 간 통신이 차단된 네트워크에서는 서버 창에 표시된 주소:포트를 직접 입력하면 됩니다.

PIN이 만료됐어요

PIN·QR의 유효시간은 120초입니다. 만료되면 서버에서 자동으로 재생성되니 새 PIN으로 다시 입력하세요. 5회 이상 잘못 입력해도 새 PIN이 발급됩니다.

연결은 됐는데 입력이 안 돼요 (macOS)

손쉬운 사용 권한이 없으면 입력이 조용히 무시됩니다. 시스템 설정 → 개인정보 보호 및 보안 → 손쉬운 사용에서 KeyU를 허용한 뒤 다시 시도하세요. 서버 창에 경고 배너가 떠 있다면 이 경우입니다.

입력이 안 돼요 (Windows)

최초 실행 시 방화벽 인바운드 허용을 거부했을 수 있습니다. Windows 방화벽 설정에서 KeyU 서버의 TCP 8770과 UDP 포트를 허용하세요.

연결이 자꾸 끊겨요

앱은 10초 주기로 세션을 확인하고, 35초 무응답이면 자동으로 재연결을 시작합니다(1초 → 2초 → 5초 간격, 최대 30초). 서버가 여러 대면 15초 후 다른 서버로 자동 전환됩니다. 절전으로 Wi-Fi가 끊기는 폰 설정도 확인해 보세요.

기억해 둘 것 — 보안은 기본값입니다

모든 통신은 로컬 네트워크 안에서만 이루어지고 전 채널이 암호화됩니다. 계정도, 클라우드도, 데이터 수집도 없습니다. PIN/QR로 직접 승인한 기기만 연결할 수 있고, 인증키는 폰의 보안 저장소와 PC 로컬 파일에만 존재합니다.

KeyU — 폰을 PC의 입력장치로 · 기술 스펙 전체는 함께 제공되는 제품 브로셔를 참고하세요.
